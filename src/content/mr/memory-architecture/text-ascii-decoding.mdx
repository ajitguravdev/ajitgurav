# **Module 5: शब्दांची आणि अक्षरांची साठवणूक (Text & ASCII Decoding)**

आतापर्यंत आपण पाहिले की Memory मध्ये केवळ आकडे (Binary Numbers) साठवले जातात आणि त्यांची बेरीज करून Value ठरते. पण मग आपल्याला प्रश्न पडतो की, **जर RAM ला फक्त आकडेच समजतात, तर ती 'A', 'B', 'C' सारखी अक्षरे (Characters) किंवा आपले नाव Memory मध्ये कसे साठवते?**

याचे उत्तर एका अत्यंत हुशार इंजिनिअरिंग संकल्पनेत दडलेले आहे — **"Encoding"** (एन्कोडिंग).

---

## **१. संगणकाला शब्द कसे समजतात? (ASCII टेबलची गरज)**

जेव्हा सुरुवातीला Computer बनवले गेले, तेव्हा Engineers समोर मोठा प्रश्न होता: अक्षरे (Characters) `0` आणि `1` च्या स्वरूपात कशी मांडायची?

यावर त्यांनी एक सोपा उपाय शोधला. त्यांनी एक 'प्रमाणित तक्ता' (Standard Table) बनवला आणि जगातील प्रत्येक अक्षराला, चिन्हाला (Symbol) आणि Space ला एक विशिष्ट 'आकडा' (Number) नेमून दिला.

या तक्त्यालाच **ASCII (American Standard Code for Information Interchange)** म्हटले जाते.

* **'A'** पासून **'Z'** पर्यंत: 65 ते 90 आकडे दिले.  
* **'a'** पासून **'z'** पर्यंत: 97 ते 122 आकडे दिले.  
* **'0'** ते **'9'** (जेव्हा ते मजकूर/Text म्हणून असतात): 48 ते 57 आकडे दिले.  
* **Space (स्पेस):** 32 हा आकडा दिला.

जेव्हा तुम्ही Keyboard वर 'A' दाबता, तेव्हा Keyboard CPU ला 'A' पाठवत नाही, तो **65** हा आकडा पाठवतो\! आणि मग CPU त्या 65 चे Binary मध्ये रूपांतर करून RAM मध्ये साठवतो.

---

## **२. मेमरीच्या आत एक अक्षर (Character) कसे दिसते?**

आपण 'A' या अक्षराचे उदाहरण घेऊ.

* **Character:** 'A'  
* **ASCII Value:** 65  
* **Binary Math:** 65 म्हणजे 64 \+ 1\.  
* **Binary Value:** `01000001`

```Plaintext
 ┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐
 │  128 │  64  │  32  │  16  │   8  │   4  │   2  │   1  │ ⟵ Place Value
 ├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤
 │   0  │   1  │   0  │   0  │   0  │   0  │   0  │   1  │ ⟵ ASCII 65 ('A')
 └──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘
```

**वरील Diagram मध्ये काय दाखवले आहे?**

* या diagram मध्ये 'A' हे अक्षर Binary मध्ये कसे साठवले जाते हे दाखवले आहे.  
* 'A' ची ASCII Value 65 आहे. त्यामुळे 64 आणि 1 च्या स्थानावरील Bits चालू (`1`) केले आहेत.  
* इतर सर्व Bits बंद (`0`) आहेत, ज्यामुळे एकूण बेरीज 65 होते.

---

## **३. गणितातील 1 (Number) आणि मजकुरातील '1' (String) मधील फरक**

हे Memory च्या लेव्हलवर दृश्यात्मक (Visually) समजून घेऊया:

**गणिताचा Number 1:** हा थेट Binary मध्ये साठवला जातो.

**मजकुराचा String '1':** हा ASCII Table नुसार साठवला जातो. (ASCII Table मध्ये '1' ची Value 49 आहे. 49 म्हणजे 32 \+ 16 \+ 1\)

```Plaintext
 ┌──────────────────┬────────────────────────────┬──────────────────┐
 │ Data Type        │ Binary Representation      │ Decimal Value    │
 ├──────────────────┼────────────────────────────┼──────────────────┤
 │ Math Number 1    │  [0][0][0][0][0][0][0][1]  │ Value = 1        │
 │ Text String '1'  │  [0][0][1][1][0][0][0][1]  │ Value = 49       │
 └──────────────────┴────────────────────────────┴──────────────────┘
```

**वरील Diagram मध्ये काय दाखवले आहे?**

* या diagram मध्ये गणितातील आकडा `1` आणि मजकुरातील (String) अक्षर `'1'` यातील फरक दाखवला आहे.  
* Number 1 थेट `00000001` म्हणून साठवला जातो.  
* String '1' ची ASCII Value 49 असल्यामुळे तो `00110001` (32 \+ 16 \+ 1\) असा साठवला जातो.

म्हणूनच JavaScript मध्ये `"1" + "1"` केल्यावर Memory मध्ये थेट बेरीज होत नाही, तर दोन Strings एकमेकांना जोडले जातात (Concatenation).

---

## **४. पूर्ण शब्द (String) मेमरीमध्ये कसा साठवला जातो?**

जेव्हा एकापेक्षा जास्त अक्षरे येतात, तेव्हा RAM मधील सलग Bytes वापरले जातात. समजा, आपण **"AJIT"** ही String Memory मध्ये साठवली, तर RAM च्या आत सलग 4 डबे (Bytes) कसे वापरले जातील ते पाहा:

* **A** \= 65 ⟵ `01000001`  
* **J** \= 74 ⟵ `01001010` (64 \+ 8 \+ 2\)  
* **I** \= 73 ⟵ `01001001` (64 \+ 8 \+ 1\)  
* **T** \= 84 ⟵ `01010100` (64 \+ 16 \+ 4\)

```Plaintext
 ┌────────────────┬───────────────────────────┬───────────────────┐
 │ Memory Address │ Data in Byte (8 Bits)     │ Character Meaning │
 ├────────────────┼───────────────────────────┼───────────────────┤
 │ 0x00000010     │ [0][1][0][0][0][0][0][1]  │ ⟵ 'A' (ASCII 65) │
 │ 0x00000011     │ [0][1][0][0][1][0][1][0]  │ ⟵ 'J' (ASCII 74) │
 │ 0x00000012     │ [0][1][0][0][1][0][0][1]  │ ⟵ 'I' (ASCII 73) │
 │ 0x00000013     │ [0][1][0][1][0][1][0][0]  │ ⟵ 'T' (ASCII 84) │
 └────────────────┴───────────────────────────┴───────────────────┘
```

**वरील Diagram मध्ये काय दाखवले आहे?**

* या diagram मध्ये **"AJIT"** हा संपूर्ण शब्द Memory मध्ये कसा बसतो हे दाखवले आहे.  
* प्रत्येक अक्षरासाठी सलग Memory Addresses (उदा. 0x10, 0x11, 0x12, 0x13) वाटून दिले आहेत.  
* प्रत्येक Address वर त्या विशिष्ट अक्षराची ASCII Binary Value साठवली आहे.

**String ची लांबी (Length) मेमरी कशी ठरवते?**

तुमच्या लक्षात आले असेल की "AJIT" या शब्दात 4 अक्षरे (Characters) आहेत. प्रत्येक अक्षरासाठी 1 Byte लागला. म्हणजेच या String ने RAM मध्ये सलग **4 Bytes (32 Bits)** एवढी जागा घेतली. तुम्ही जेवढी मोठी String लिहाल, तेवढे जास्त सलग Bytes मेमरीमध्ये राखून ठेवले जातील.

---

आणि हो, याच 0 ते 255 च्या रेंजमध्ये (ASCII टेबलनुसार) इंग्रजीची सर्व मोठी अक्षरे (A-Z), छोटी अक्षरे (a-z), आकडे (0-9) आणि चिन्हे (Symbols) आरामात सामावतात. म्हणूनच 'Text' साठवण्यासाठी 8 बिट्सची (1 Byte ची) ही साईझ एकदम अचूक आणि पुरेशी ठरते.

---

### **💡 Technical Key Takeaway**

**ASCII Encoding** हा तो दुवा (Bridge) आहे जो मानवी भाषेला (Text/Characters) मशीनच्या भाषेत (Binary Numbers) जोडतो. Memory मध्ये String साठवताना, प्रत्येक Character ला त्याच्या ASCII Value नुसार एका स्वतंत्र **Byte** मध्ये सलगपणे साठवले जाते. म्हणूनच String ची Length जेवढी मोठी, तेवढे जास्त Bytes Memory मध्ये वापरले जातात आणि JavaScript मध्ये String Concatenation याच आधारावर काम करते.
