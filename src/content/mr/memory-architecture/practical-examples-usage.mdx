# **Module 3: प्रत्यक्ष उदाहरणे आणि वापर (Practical Examples & Usage)**

आतापर्यंत आपण बायनरी गणिताचे नियम पाहिले. पण मेमरीच्या आत या Values प्रत्यक्ष कशा तयार होतात, याचा कधी विचार केला आहे का? चला, याची काही अत्यंत सोपी आणि रंजक उदाहरणे पाहूया.

---

## **१. बायनरी उदाहरणे: आकडे कसे तयार होतात? (Binary Examples)**

### **उदाहरण १: '1' हा आकडा कसा तयार होतो?**

तुम्हाला जर मेमरीमध्ये `1` हा आकडा Store करायचा असेल, तर काय करावे लागेल? अतिशय सोपे आहे\! फक्त सर्वात उजवीकडचा (ज्याची किंमत 1 आहे) तो स्विच चालू (ON) करायचा.

| Plaintext  ┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐  │  128 │  64  │  32  │  16  │   8  │   4  │   2  │   1  │ ⟵ Place Value  ├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤  │   0  │   0  │   0  │   0  │   0  │   0  │   0  │   1  │ ⟵ Bits (ON \= 1\)  └──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘                                                        Total Value \= 1 |
| :---- |

**वरील Diagram मध्ये काय दाखवले आहे?**

* या diagram मध्ये 1 Byte मधील 8 Bits दाखवले आहेत.  
* 1 स्थानाचा Bit हा `1` (ON) अवस्थेत आहे.  
* बाकीचे सर्व Bits हे `0` (OFF) अवस्थेत आहेत.  
* त्यामुळे ON असलेल्या स्थानाची बेरीज केल्यास एकूण Value ही `1` होते.

### **उदाहरण २: '5' हा आकडा कसा तयार होतो?**

आता जर तुम्हाला `5` हा आकडा बनवायचा असेल, तर कसे कराल?

गणितानुसार, 5 म्हणजे 4 \+ 1\. त्यामुळे आपण फक्त '4' आणि '1' किंमत (Place Value) असलेले स्विचेस चालू (ON) करू.

| Plaintext  ┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐  │  128 │  64  │  32  │  16  │   8  │   4  │   2  │   1  │ ⟵ Place Value  ├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤  │   0  │   0  │   0  │   0  │   0  │   1  │   0  │   1  │ ⟵ Bits (4 \+ 1\)  └──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘                                                        Total Value \= 5  |
| :---- |

**वरील Diagram मध्ये काय दाखवले आहे?**

* या diagram मध्ये 1 आणि 4 या स्थानांचे Bits हे `1` (ON) आहेत.  
* बाकीचे सर्व Bits बंद (`0`) आहेत.  
* त्यामुळे, ज्या स्थानांचे Bits ON आहेत त्यांची बेरीज केल्यास (4 \+ 1 \= 5\) आपल्याला एकूण Value `5` मिळते.

## **२. 0 ते 255 चा प्रवास (The Magic of 255\)**

तुम्हाला नेहमी प्रश्न पडत असेल की, एका Byte मध्ये फक्त 255 पर्यंतच आकडे का मावतात? याचे उत्तर याच बायनरी बेरजेत दडलेले आहे.

जर आपण 8 च्या 8 स्विचेस चालू (ON) केले, तर काय होईल? तेव्हा त्या सर्व जागांच्या किमतींची (Place Values) बेरीज होते.

| Plaintext  ┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐  │  128 │  64  │  32  │  16  │   8  │   4  │   2  │   1  │ ⟵ Place Value  ├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤  │   1  │   1  │   1  │   1  │   1  │   1  │   1  │   1  │ ⟵ All Bits ON  └──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘                                                        Total Value \= 255  |
| :---- |

**वरील Diagram मध्ये काय दाखवले आहे?**

* या diagram मध्ये 1 Byte मधील सर्व 8 स्विचेस `1` (ON) अवस्थेत दाखवले आहेत.  
* **गणित:** $128 \+ 64 \+ 32 \+ 16 \+ 8 \+ 4 \+ 2 \+ 1 \= 255$  
* जेव्हा CPU या 8 Bits ला 1, 2... 128 ही स्थानाची किंमत लावतो, तेव्हा जर सर्व स्विचेस चालू असतील, तर त्यांची एकूण बेरीज जास्तीत जास्त 255 एवढीच येऊ शकते.

यापेक्षा मोठा आकडा (उदा. 256\) या एका Byte च्या डब्यात बसूच शकत नाही, कारण जागा संपल्या आहेत\! 256 साठवण्यासाठी आपल्याला 9 वा डबा (किंवा दुसरा Byte) जोडावा लागतो, जे आपण पुढच्या मॉड्यूल्समध्ये पाहणार आहोत.

## **३. True आणि False ची साठवणूक (Hardware-Level Booleans)**

Programming मध्ये आपण नेहमी `true` आणि `false` (Boolean Values) वापरतो. पण Hardware मध्ये सत्य आणि असत्य (Truth and Lie) अशी कोणतीही संकल्पना नसते हे तुम्हाला माहीत आहे का? तिथे फक्त "शून्य" (Zero) आणि "शून्य नसलेली Value" (Non-Zero) एवढेच असते.

1 Byte (8 Bits) च्या डब्यात हे कसे साठवले जाते, ते पाहा:

### **False (असत्य) \= सर्व स्विचेस OFF**

जर डब्यातील आठही स्विचेस बंद (OFF) असतील, तर ती Value पूर्णपणे शून्य (Zero) असते. CPU च्या आतील 'Zero Flag' ॲक्टिव्हेट होतो आणि System याला **False** मानते.

| Plaintext  ┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐  │  128 │  64  │  32  │  16  │   8  │   4  │   2  │   1  │ ⟵ Place Value  ├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤  │   0  │   0  │   0  │   0  │   0  │   0  │   0  │   0  │ ⟵ All OFF  └──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘                                                        Result \= FALSE (0)  |
| :---- |

**वरील Diagram मध्ये काय दाखवले आहे?**

* येथे सर्व 8 Bits ची अवस्था `0` (OFF) आहे.  
* कोणतीही बेरीज होत नाही, त्यामुळे एकूण Value `0` आहे, ज्याला Hardware **False** मानते.

### **True (सत्य) \= किमान 1 तरी स्विच ON**

जर या 8 डब्यांपैकी एक जरी डबा `1` झाला (म्हणजे करंट चालू झाला), तर ती Value आता शून्य (Zero) राहत नाही. कोणतीही Non-Zero Value हार्डवेअरला **True** वाटते.

साधारणपणे, C++ किंवा System Level वर True दर्शवण्यासाठी सर्वात उजवीकडचा (Bit 0\) चालू (ON) केला जातो.

| Plaintext  ┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐  │  128 │  64  │  32  │  16  │   8  │   4  │   2  │   1  │ ⟵ Place Value  ├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤  │   0  │   0  │   0  │   0  │   0  │   0  │   0  │   1  │ ⟵ At least 1 ON  └──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘                                                        Result \= TRUE (1) |
| :---- |

**वरील Diagram मध्ये काय दाखवले आहे?**

* येथे किमान एक Bit (सर्वात उजवीकडचा) `1` (ON) अवस्थेत आहे.  
* त्यामुळे एकूण Value `1` (Non-Zero) होते, ज्याला Hardware **True** मानते.

---

### **💡 Technical Key Takeaway**

थोडक्यात सांगायचे तर: **RAM** फक्त `0` आणि `1` चा साठा करते. पण जेव्हा **CPU** ते वाचतो, तेव्हा तो आपल्या बायनरी गणिताच्या (Binary System) नियमानुसार त्यांना 1, 2, 4, 8 अशी किंमत (Place Value) देतो आणि त्यातून खरा अर्थ (Value) काढतो\! Hardware-level वर `0` म्हणजे **False** आणि कोणतीही Non-Zero Value म्हणजे **True** मानली जाते.
