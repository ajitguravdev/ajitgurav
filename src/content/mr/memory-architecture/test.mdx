## ३-बिट्सची जादू

येथे तुम्हाला ३-बिट्सची जादू दिसेल. जेव्हा तुम्ही ३ वायर्स वापरता, तेव्हा तुम्ही एकूण ८ (२^३) वेगवेगळ्या गोष्टी कंट्रोल करू शकता!

</magic-box>

येथे बॉक्सच्या बाहेरचा मजकूर आहे.

Software Developers नेहमी 'High-Level' Programming Languages मध्ये (जसे JavaScript) काम करतात. तिथे आपण सहज `array[5]` असे लिहितो. पण Hardware Level वर '5' नावाचा कोणताही आकडा नसतो...

<lesson-section title="१. बायनरीचे खरे गणित (The 4-2-1 Rule)" icon="calculator">

Memory मध्ये जेव्हा अनेक Bits एकत्र येतात, तेव्हा उजवीकडून डावीकडे जाताना प्रत्येक Bit ची क्षमता दुप्पट होत जाते.

---

```diagram-binary
┌─────────────────┬─────────┬─────────┬─────────┐
│ Place Value     │   [4]   │   [2]   │   [1]   │
├─────────────────┼─────────┼─────────┼─────────┤
│ Hardware Switch │   ON    │   OFF   │   ON    │
│ Binary State    │    1    │    0    │    1    │
├─────────────────┴─────────┴─────────┴─────────┤
│ System Calc     │ (4x1) + (2x0) + (1x1) = 5   │
└───────────────────────────────────────────────┘
```

<info-box>

- या diagram मध्ये एक 3-to-8 Decoder Architecture दाखवले आहे, ज्याला 3 Inputs (A, B, C) आणि 8 Outputs (Wire 0 ते 7) आहेत.
- Input म्हणून `101` (म्हणजेच 5) हा पॅटर्न पाठवला आहे.
- Logic Gates हा पॅटर्न 'Decode' करून फक्त आणि फक्त Wire 5 (TV) मध्ये करंट पाठवतात, बाकी सर्व Wires बंद राहतात.
  
</info-box>

</lesson-section>

<takeaway-box>
निष्कर्ष: Computer मध्ये कोणताही 'जादूचा शोध' (Magic Search) नसतो. Memory मधील कोट्यवधी डब्यांमधून एक अचूक Data शोधणे, हे पूर्णपणे 'Logic Gates' आणि 'Binary Decoding' ची भौतिक आणि गणितीय प्रक्रिया आहे. 0 आणि 1 चे पॅटर्न केवळ Data साठवत नाहीत, तर ते 'कोणत्या रस्त्याने जायचे' याचे नकाशे (Routing Logic) म्हणूनही काम करतात.
</takeaway-box>
